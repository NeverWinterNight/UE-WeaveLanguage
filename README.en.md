## Weave Language

A graph layout language optimized for AI, currently used in Blueprint AI.

Blueprints have a lot of functionality, but escaping is`K2Node`text, right`LLM`It's quite a burden.  
`Weave语言`It's people and`LLM`All can read and write blueprints.

This plug-in aims to open up the interaction barriers between LLM and Blueprint, and create middleware for the interaction between AI and Blueprint. In the future, it will gradually support material charts, animation charts, Control Rig charts, etc. We hope that developers can work together to develop it.

### characteristic

-   **cross blueprint**: You can edit the blueprint without opening it
-   **readability**: Both humans and AI can understand
-   **bidirectional conversion**: Lossless conversion between Blueprint and Weave (under update)
-   **Patch mechanism**: AI only needs to say "change where" instead of rewriting the entire file.
-   **Schema constraints**: It is impossible for AI to generate non-existent nodes or illegal connections
-   **Token efficient**: Saves time compared to directly reading blueprint data`70%以上`token

### Weave Language Syntax Reference

one`.weave`The file contains multiple declaration blocks:

    graphset <名称> <蓝图路径>
    graph <函数名>

    node ...
    link ...
    set ...

* * *

| Keywords   | use                                                      |
| ---------- | -------------------------------------------------------- |
| `graphset` | Declare the blueprint resource, where is this blueprint? |
| `graph`    | Declare event graph/function, which specific graph is    |
| `node`     | declare node                                             |
| `link`     | Connect node endpoint                                    |
| `set`      | Set node attribute value                                 |

* * *

#### Example

##### After starting the game, use Sequence to execute four Print Strings in sequence

-   At this time, the event graph of the /Game/MyActor2 blueprint is operated.

```weave
graphset MyActor2 /Game/MyActor2.MyActor2
graph EventGraph

node beginplay : event.Actor.ReceiveBeginPlay @ (-144, 16)
node sequence : special.Sequence @ (96, 16)
node print01 : call.KismetSystemLibrary.PrintString @ (384, 16)
node d : call.KismetSystemLibrary.PrintString @ (384, 160)
node e : call.KismetSystemLibrary.PrintString @ (384, 304)
node f : call.KismetSystemLibrary.PrintString @ (384, 448)

link beginplay.then -> sequence.execute
link sequence.then_0 -> print01.execute
link sequence.then_1 -> d.execute
link sequence.then_2 -> e.execute
link sequence.then_3 -> f.execute

set print01.WorldContextObject = self
set print01.InString = Print 1
set print01.bPrintToScreen = true
set print01.bPrintToLog = true
set print01.TextColor = (R=0.000000,G=0.660000,B=1.000000,A=1.000000)
set print01.Duration = 2.000000
set d.WorldContextObject = self
set d.InString = Print 2
set d.bPrintToScreen = true
set e.WorldContextObject = self
set e.InString = Print 3
set f.WorldContextObject = self
set f.InString = Print 4
```

### How to use?

Pull the plug-in source code and put it into Unreal`cpp`In the plug-in folder of the project, regenerate the project file and start it after compilation.
You can also download the release version in Release and put it directly into the project plug-in folder (if there is one).

#### quick test

If the plug-in is successfully installed, there should be a`Weaver`drop-down menu with the word , click after opening the menu`Weave 生成解释调试器`to open. Select the blueprint node and click`Generate from Selection`You can translate the entire selected blueprint chain into Weave language. Of course, click`Apply to Blueprint`The corresponding chart can be applied.

# Weave Language API Reference

All operations are called through UWeaveOperator.
You need to call first`GenerateWeaveLanguage`Because this will cause the plugin to recalculate all events, functions, macros, types, etc., Weave needs to use the Schema generated by this function for reliable parsing. This function is called once by default after the editor is started.

### Get Blueprint Weave Code

FString GetBlueprintWeave(BlueprintPath, GraphName, EntryNode = TEXT(""))

Convert the blueprint to Weave language.

parameter:

-   BlueprintPath: Blueprint path, such as /Game/MyActor.MyActor
-   GraphName: function name or event name, such as TestFunction
-   EntryNode: optional, specify the entry node

Returns: Weave code string.

* * *

### Apply Diff

FString ApplyWeaveDiff(OriginalWeave, DiffCode, OutError)

Applies a Diff to the Weave code, returning the complete modified Weave.

parameter:

-   OriginalWeave: Original Weave code
-   DiffCode: AI generated Diff
-   OutError: error message, empty when successful

Returns: the complete modified Weave code.

* * *

### Verify Diff

FString DiffCheck(BlueprintPath, GraphName, DiffCode, OutError)

Preview Diff effects before actually applying them without modifying the Blueprint.

parameter:

-   BlueprintPath: blueprint path
-   GraphName: function name
-   DiffCode: AI generated Diff
-   OutError: error message

Returns: the previewed Weave code.

* * *

### Apply to blueprint

bool ApplyWeaveToBlueprintWithUndo(WeaveCode, BlueprintPath, GraphName, OutError)

Write Weave code into the blueprint to support undo.

parameter:

-   WeaveCode: Complete Weave code
-   BlueprintPath: blueprint path
-   GraphName: function name
-   OutError: error message

Return: true indicates success.

* * *

void GenerateWeaveLanguage()
